How it works

For the most basic validation to work, you must design your forms as mentioned in the docs. See an example below:

The .form-group parent element

In the above example, all the input fields are properly written using Bootstrap's guideline i.e. each input field have a parent element with class .form-group. This library looks up for this parent element for each input fields to validate and inserts the error message accordingly and adds the .has-error class to that element.

The bs-form-group parent element directive

If your form is designed in a different way which does not use the .form-group as a parent element's class, then optionally you can add a bs-form-group attribute to a parent element for each input element to tell this library that this particular element with attribute bs-form-group is a parent for this input element.

The bs-validation directive

Another important thing to note here that all the input elements which requires validation must have bs-validation directive applied and must be using the ng-model directive.

What?? bs-validation directive in each input field

Yes, bs-validation is the main directive which needs to be added to each input element in a form in order to enable the validation. But, you don't see that in the above example?? Continue reading..

The form directive

That is the magic! If your all the input elements are wrapped inside a form element then you don't have to insert bs-validation directive to all your input elements. This library will do it for you automagically using a form directive available in this module. This will save you from the tedious work of adding this directive to all the input elements if you have say 30 input elements in a form.

Validation state

There are currently two types of validation state available:
  1. Success: i.e. when a certain input element passes all the validations
  2. Error: i.e. when a certain input element fail to pass minimum of one validation

Success validation state

Whenever an input element is marked as valid (i.e. it passed all the validation), then the .has-success class is added to the form group parent element as described in the forms control validation section in the Bootstrap.

Error validation state

Whenever an input element is marked as invalid (i.e. it fail to pass any of the the validation), then the .has-error class is added to the form group parent element as described in the forms control validation section and also the default configured message will be displayed as the block-level help text below in the input field. 

<span src="help-block bs-invalid-msg">This field is required.</span>

The class .has-error can be customized in the app's config. See the configuration and example section for this. This is the simple way of displaying the error message. This module also provide another type of displaying the error message as tooltip. Check the next section for this.

Events for validation state

The actual validation is handled by the Angular itself all the time. This module just displays the validation state on various events. The default event of displaying validation state is blur i.e. the validation state will be shown as the user lost focus from an input field. Validation events available are:

  1. blur: Display validation state as the input is blurred i.e. focus is lost from that input
  2. submit: Display the validation after the form submission is attempted
  3. display: Display the validation state as the form is presented to the user

Any combination of these events can be customized globally. Check the next section for the usage.

The validator

This module uses the $validator introduced in Angular 1.3.0 to provide validation and watches for the input field validity to show/hide the validation state.
