Changing the event for displaying validation state

The below example will now display the validation state only after user attempt to submit the form. A few things to look out for when playing around this particular example:

  1. Before a field is marked as invalid, the validation is lazy: Before submitting the form for the first time, the user can tab through fields without getting annoying messages – they won't get bugged before having the chance to actually enter a correct value.

  2. Once a field is marked invalid, it is eagerly validated: As soon as the user has entered the necessary value, the error message is removed.

  3. If the user enters something in a non-marked field, and tabs/clicks away from it (blur the field), it is validated – obviously the user had the intention to enter something, but failed to enter the correct value.

  4. After trying to submit an invalid form, the first invalid element is focused, allowing the user to correct the field.

This validation strategy is inspired from the jQuery validation mechanism.

Customizing messages & in-build validations

The following example demonstrate how can you override the default messages per input field and per error key. Also explains some built-in validators. You can try the following:

  • First try to submit the form without filling in. This will display all the validation state.
  • The message for required validation on the email field is changed by setting required-notification="My message" attribute to the input field where required was the validation error key name.
  • Try entering some value in the first name and then try inputting some numbers into it. That will display the default validation message for pattern validation.
  • Do the above for last name and see it's message. Message is overridden by pattern-notification="My message" attribute.
  • Enter a different value for the password and confirm password. The confirm password field will complain that value does not match. This is achieved by an in-built validator equalto="" which accepts the ng-model name of other field.
  • Another in-built validation from this library is applied to the age field i.e. min validation to validate a minimum value.

Manually showing/hiding the validation state

Each ngModelController & formController having a utility method to manually show/hide the validation which is helpful at certain cases. To show validation message, use $showValidation() and to hide validation messgae, use $hideValidation() method.

Using tooltip instead

All the above example uses the simple mechanism of displaying error messages. This module supports additional option to display error messages as tooltip.

This module has a dependency over the Angular's ui-bootstrap library. To be more specific, only $uibPosition service is required to display the tooltip. Checkout the example below:

All the above configuration will also be applicable for the tooltip based error messages. The default position of the tooltip is set to bottom-left. Which can be customized globally as well as per form/input element. Look at the below example to see customization.

  • Display type can be configured per specific input element or to specific form using bs-display-error-as="tooltip" or bs-display-error-as="simple"
  • If using tooltip type error display, the tooltip placement can be configured per specific input element or to specific form using bs-tooltip-placement="right". Placement values can be one of the following as specified in the docs:
    • top: element on top, horizontally centered on host element.
    • top-left: element on top, left edge aligned with host element left edge.
    • top-right: element on top, right edge aligned with host element right edge.
    • bottom: element on bottom, horizontally centered on host element.
    • bottom-left: element on bottom, left edge aligned with host element left edge.
    • bottom-right: element on bottom, right edge aligned with host element right edge.
    • left: element on left, vertically centered on host element.
    • left-top: element on left, top edge aligned with host element top edge.
    • left-bottom: element on left, bottom edge aligned with host element bottom edge.
    • right: element on right, vertically centered on host element.
    • right-top: element on right, top edge aligned with host element top edge.
    • right-bottom: element on right, bottom edge aligned with host element bottom edge.
    • auto: automatically find the best fit for the placement.
